# InSituTutor MVP Scope

## 1. 做什么（In Scope）

### 1.1 业务能力

- 使用 `Qwen3.5-Omni` 对老师示教长音视频进行结构化分析
- 构建任务级知识结构：`steps/actions/attention_cues/errors/interventions`
- 学生练习时进行实时步骤提示（当前步骤 + 下一步）
- 使用简化规则进行偏差检测并触发纠偏提示
- 全链路日志记录，支持后续复盘

### 1.2 产品形态

- 仅学生端实时界面（网页示意）
- 提示输出为文本（可扩展语音）
- 后续可将网页提示映射到投影显示

### 1.3 技术策略

- 优先规则引擎，暂不引入复杂在线模型推理
- 先保证单任务可跑通，再考虑跨任务迁移
- 优先可解释性（每条提示可回溯到规则或知识点）

## 2. 不做什么（Out of Scope）

- 不做教师端控制台产品化
- 不做复杂机器人控制、自动执行闭环
- 不做高精度细粒度动作打分（如毫米级姿态纠正）
- 不做多任务通用平台能力（账号体系、多租户、权限系统）
- 不做云端大规模部署与成本优化

## 3. MVP 验收口径

- 至少 1 个任务可完整跑通
- 每个任务至少 5 个步骤
- 每步至少包含 1 个观测点、1 个常见错误、1 个纠偏策略
- 至少 3 类简单偏差可检测并提示
- 偏差到提示平均延迟目标 < 2 秒
- 日志可追踪“偏差 -> 提示 -> 是否纠正成功”
