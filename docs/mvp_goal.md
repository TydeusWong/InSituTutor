# InSituTutor MVP Goal Definition

## 1) MVP 目标

在智慧空间中，完成最小可运行闭环：

1. 老师示教视频输入系统（离线）
2. AI 从示教中提取结构化教学知识（步骤、关键动作、观测点、常见错误、纠偏方式）
3. 学生练习时，系统基于当前步骤给出实时提示（网页示意）
4. 学生发生偏差时，系统在低延迟下触发纠错提示

一句话定义：**让学生端在真实练习中“看得懂下一步、犯错能被及时提醒”。**

## 2) MVP 边界（做 / 不做）

### 做

- 只做学生端实时引导界面（MVP 先网页，后续可接投影）
- 只做“提示与纠偏”，不做机器人控制与自动执行
- 偏差检测先用简化逻辑（顺序、存在性、基础姿态/相对位置规则）
- AI 分析模型固定为 `Qwen3.5-Omni`（长音视频分析）

### 不做

- 不做教师端产品化控制台
- 不做复杂动作控制、复杂规划、端到端自主决策
- 不做跨任务通用大规模泛化，只保障单任务可跑通

## 3) 输入 / 输出定义

### 输入

- 老师示教视频（长视频，含语音讲解）
- 学生练习实时视频流（摄像头）
- 任务配置（任务名称、所需物体、场景约束）

### 输出

- 结构化教学知识（`steps/actions/attention_cues/errors/interventions`）
- 学生端实时提示（当前步骤、下一步建议）
- 偏差告警与纠偏提示（命中规则时触发）
- 过程日志（步骤推进、偏差触发、纠错结果）

## 4) 成功指标（MVP 验收口径）

为保证可落地，MVP 先设“工程可达”指标：

- **闭环可运行**：至少 1 个任务可端到端跑通（示教分析 -> 实时提示 -> 偏差纠错）
- **结构化完整度**：每个任务至少 5 个步骤；每步至少 1 个观测点 + 1 个错误 + 1 个纠偏策略
- **偏差检测能力**：至少支持 3 类简单偏差（顺序错误、对象缺失、姿态/相对位置粗偏差）
- **实时性**：从观测到偏差到提示触发，平均延迟目标 < 2 秒
- **可复盘性**：日志可追踪“偏差 -> 提示 -> 是否纠正成功”

## 5) 第 0 步完成定义（Definition of Done）

当以下条件同时满足，即视为第 0 步完成：

1. 目标、边界、输入输出、指标均形成文档并冻结
2. 团队对“只做学生端网页示意 + 后续投影”的产品形态达成一致
3. 团队对“`Qwen3.5-Omni` 为示教分析模型”达成一致
4. TODO 清单中第 0 步勾选完成，并可追溯到本文件
